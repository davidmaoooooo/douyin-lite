### s4 解码的块对齐（【已确定】，可手算核验）

## 三、RC4 已知明文攻击的可行性分析（【已确定】方法论）

### 核心原理
RC4 keystream 只由 **key** 决定（与明文无关）。抖音 RC4 key 是常量（不随请求输入变），
因此 **所有样本共享同一条 keystream**。于是对任意两样本 A、B：

```
cipher_A[i] ^ cipher_B[i] = plain_A[i] ^ plain_B[i]   （对所有 i 成立）
```

### 能做什么（【已确定】可行）
- **query 差分**：q=1 vs q=2 在字节 24..35 不同、其余相同。
  → 在这些位置 `cipher^cipher` = 两个 query 的双 SM3 派生明文之异或。
  可验证"这 12 字节确实来自 query 的 SM3"假设（拿 q=1/q=2 的 url_search_params
  在本地算双 SM3，看差分是否匹配）。
- **time 差分**：定位时间戳字节的明文偏移与字节序（大端/小端、是否 /256^4）。

### 不能直接做什么（【已确定】限制）
- **无法仅凭这些样本恢复 key 或完整 keystream**。原因：
  恢复 keystream[i] 需要 **已知某个样本在位置 i 的明文 plain[i]**。
  目前唯一可能已知的明文是 random 头（若它=旧 generate_random_str(0.5)）。
  但 random 头**也被 RC4 加密**，且 random 一变明文全变 —— 我们没有"明文已知且固定"的锚点。
- 旧锚点（44/6383/6241）解不出，说明要么 key 变、要么模板字节布局变、要么两者都变。
  **不能假设旧 b[] 布局照搬**。

### 恢复 key 的真正路径（需要新样本，见第五节）
要把 RC4 key 钉死，最干净的办法是**已知明文攻击**：
构造一个我们能完全预测明文的样本，使其与基线只差"已知的若干字节"，
则 `keystream[i] = cipher[i] ^ known_plain[i]`，逐字节恢复 keystream，
再由 keystream 反推 key（RC4 KSA 可在短 key 时暴力/分析恢复）。

## 五、需要采集的新样本清单（精确，请用浏览器回填）

固定基线：`Date.now=1700000000000, Math.random=0.5, performance.now=12345, 默认UA, base_url 同 _meta`。

### A. 钉死 RC4 key —— 已知明文攻击（最高优先级）
目标：制造"明文只差已知字节"的样本对。最佳手段是只改 **一个已知会进入明文且字节可预测** 的输入。
1. **Arguments 维度**（旧版 b[26..37] 来自 Arguments=[0,1,14]，字节完全可预测）：
   若新版仍有 Arguments 入口，分别采集 `Arguments=[0,1,14]`（基线）与 `[0,1,8]`、`[0,1,0]`、`[0,2,14]`。
   这几组明文差是**已知的小整数大端字节**，可直接 `cipher^known_diff` 恢复对应位置 keystream。
   —— 需要你确认新版 a_bogus 是否暴露 Arguments 入口；若不暴露，跳过。
2. **aid 维度**：分别用 `aid=6383`（基线）与 `aid=1128`、`aid=2906` 三个不同 aid 发请求采集。
   aid 字节明文已知，可定位 aid 在明文中的偏移并恢复该处 keystream。

### B. 验证 query 明文 = 双 SM3（中优先级）
3. 采集 query = 完整真实串（如基线那条长 url_search_params）与其"改 1 字符"版本各 1 条。
   用于把字节 24..35 与本地双 SM3 结果对齐核验。

### C. time 字节序定位（中优先级）
4. 已有 +0/+1/+256/+65536 够用，可选补 `+16777216`(2^24)、`+4294967296`(2^32) 各 1 条，
   用于确认时间戳是否拆成 lo32 大端 + 高位 /256^4 /256^5（旧版 b[20..25]/b[44..50] 写法）。

### D. random 头长度与边界（低优先级）
5. 采集 Math.random = `0.123456789`、`0.999999` 各 1 条（其余冻结），
   配合 0.0/0.25/0.5/0.75 判断 random 头占明文前几字节、是否仍是旧 gener_random 12 字节布局。
